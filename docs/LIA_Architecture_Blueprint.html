Intelligence engine
claude-sonnet-4-6 model. Same API key as existing tools. ~$0.05–0.15 per output generation. Max tokens 8000 for rich output.

GitHub private repo
Context store
Structured /context folder. Subfolders: /observe /absorb /reflect /decisions. Read via GitHub API. Version controlled. Team can contribute.

Design principles
What governs every build decision
These principles were established through the product design conversation and apply to every session.
1
Nothing is hardcoded Every output is generated dynamically from context. There are no templates with placeholder fields. LIA reasons about what to include, in what order, with what emphasis — every time.
2
Audience-aware always The same week's context produces a completely different document for a CIO versus a direct report versus a board. LIA knows who is reading and adapts accordingly.
3
Zero friction capture If it takes more than 90 seconds to add something to LIA's context, it won't happen. Every input mechanism is designed around the busiest executive day imaginable.
4
Judgment over transcription LIA captures how the leader sees the world, not just what was said. A voice observation after a meeting is more valuable than a transcript because it includes interpretation.
5
Daily reflection, not weekly Weekly tools get forgotten. Daily tools become habits. LIA checks in every day with 3 smart questions based on what happened. 5 minutes. Compounding forever.
6
No IT approval required LIA is a local HTML file calling an API. Nothing touches the corporate network. No software install. No security review. Works anywhere, day one.
